# Decision 0002: Create CIB/165-Facing Threads Report v0 By 2026-04-30

## Date

2026-04-23

## Decision

Create a real initial research report, `reports/threads-scam-content-research-v0.md`, for CIB/165-facing review by 2026-04-30.

The report must be Threads-only and must move beyond concept framing into a concrete research plan: scope, dataset v0 design, annotation plan, baseline comparison, evaluation framework, budget-fit argument, governance gates, and four-week execution path.

## Context

Morning CIB/165 information and stakeholder discussion shifted the immediate deliverable. The project should no longer stop at a stakeholder scoping memo. The next artifact should be a readable v0 initial research report that can anchor review and narrowing.

## Options Considered

| Option | Assessment |
|---|---|
| Keep producing a concept/scoping memo | Too weak for the April 30 deadline and does not turn the scaffold into an actionable research package. |
| Build a prototype first | Premature without approved data, stable labels, and a report-level scope agreement. |
| Produce a Threads report v0 first | Best fit: concrete enough for stakeholders, honest about missing data, and aligned with the docs-first repo posture. |

## Rationale

The project needs a defensible external-facing shape before data collection and experiments expand. A v0 report can state the CIB/165 context, define what is and is not in scope, and lock the first experiment without overpromising production detection.

## Consequences

- `reports/` becomes a first-class repo area.
- The April 30 v0 report is now the immediate research deliverable.
- The first dataset and baseline work should support the report's plan, not drift into unrelated implementation.
- The planning repo should track the deadline and agenda blocks.
- Google Calendar should contain focus blocks and a deadline marker.

## Follow-Up

- Complete report assembly by 2026-04-30.
- Revisit scope after the report is reviewed.
- Do not collect real data until governance approval is recorded.
