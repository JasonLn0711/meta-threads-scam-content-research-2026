# Meta System

This directory contains durable v2 research-operating-system metadata:

- `sparse_schema/` defines the explainable sparse feature layer.
- `feature_candidates/` stores proposed emergent features generated from discrepancy analysis.
- `feature_review_queue/` stores human review queues for accepting or rejecting proposed features.

Do not store raw Threads content, personal identifiers, screenshots, URLs, contact handles, browser artifacts, credentials, or controlled-store paths here.
