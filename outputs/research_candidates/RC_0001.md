# RC_0001: Review-Stable Funnel Anchor As First Learning-Loop Feature

## Observation

The first v2 discrepancy loop found one `missed_pattern`: `synthetic-candidate-0001` and `synthetic-candidate-0002` were close in discovery-only embedding space but were not grouped by sparse features.

After human review, the shared abstraction was not raw text or identity evidence. It was a metadata-level pattern:

```text
funnel-stage or value-stage anchor + low review cost + no second-review requirement
```

This was encoded as `review_stable_funnel_anchor`.

## Evidence

- Source discrepancy: `disc-0001`
- Feature candidate: `meta-system/feature_candidates/review_stable_funnel_anchor.yaml`
- Human queue decision: `accepted`
- Active schema: `meta-system/sparse_schema/sparse_features_v2.yaml`
- Batch: `batch-0003-dual-track-dry-run`

## Effect

Sparse clustering changed:

- Before: `synthetic-candidate-0001` clustered with `synthetic-candidate-0003`.
- After: `synthetic-candidate-0001` clustered with `synthetic-candidate-0002`.
- New shared core feature: `review_stable_funnel_anchor`.

Discrepancy changed:

- `missed_pattern`: `1 -> 0`
- `over_generalization`: `1 -> 0`

SVS changed:

- `review_stable_funnel_anchor` entered the ranking with `svs: 0.004473953`.
- Batch SVS increased from `0.001053825` to `0.00112473`.
- Reply/contact family signals decreased slightly, which suggests the new feature separated review-stable funnel anchors from thread-dependent ambiguous cases.

Review cost did not actually decrease inside this fixed synthetic batch. The useful signal is that the new feature isolates a lower-burden subgroup with average review time `105.0` seconds versus the batch average `116.25` seconds.

## Hypothesis

High-value candidate discovery improves when the sparse layer distinguishes review-stable funnel anchors from broader reply/thread-dependent funnel cues.

This may help route reviewer effort toward candidates that combine useful discovery yield with lower review burden, without using embedding outputs as decisions.

## Next Experiment

Run the same learning loop on a larger metadata-only batch and check whether `review_stable_funnel_anchor` remains stable across:

- reply-funnel candidates;
- guarantee/result-display candidates;
- hard-negative or uncertain candidates;
- second-review-required cases.

Continue only if the feature improves cluster cohesion and SVS without hiding hard-negative ambiguity or leaking final-label logic into pre-review candidate discovery.
